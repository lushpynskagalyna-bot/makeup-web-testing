1. Introduction

This document describes the testing strategy, scope, and approach for manual testing of the MAKEUP e-commerce website.
The purpose of this test plan is to ensure that the main website functionality works as expected and provides a positive user experience.

2. Objective

The objective of testing is to verify that the core features of the website function correctly according to expected user behavior and business requirements.

3. Scope of Testing
In Scope:

Homepage

Search functionality

Product details page

Add to cart / Remove from cart

Checkout process

Negative scenarios (invalid inputs, out-of-stock cases, etc.)

Browser console monitoring (error checking via DevTools)

Out of Scope:

Payment gateway integration

Performance testing

Security testing

Mobile testing

4. Test Types

Functional Testing

Negative Testing

UI Testing

Exploratory Testing

Smoke Testing

5. Test Environment

Browser: Google Chrome

Tools: Chrome DevTools, Google Sheets

Operating System: Windows

Testing Type: Manual Testing

6. Entry Criteria

Website is accessible

Core functionality is implemented

Test cases are prepared

7. Exit Criteria

All planned test cases are executed

Critical and high-priority defects are documented

Test results are summarized

8. Risks

Website functionality may change during testing

Environment instability

Limited access to backend functionality
